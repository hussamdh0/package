get /journey

Get list of journeys

Fields: id, name, origin, destination, phone, email

id: primary key

phone & email: each journey can have its contact info, but if those don't exist, the contact of the journey user is returned

  • Request
  • Response

Query Parameters

  • date: (string)

    return only journeys on that date or.. (see date_tolerance)

    Example:

    2019-12-25
  • date_tolerance: (integer)

    should be accompanied with param 'date' and specifies how tolerant filtering dates is

    Example:

    1
  • origin: (int (city id))

    return journey starting within 'radius' away from

    Example:

    1941 (Dresden)
  • destination: (int (city id))

    return journey arriving within 'radius' away from

    Example:

    2059 (Cairo)
  • radius: (int (km))

    explained, default = 50

    Example:

    1
  • my_journeys: (boolean)

    return journeys of logged in user only (ignored if no one is logged in)

    Example:

    1
